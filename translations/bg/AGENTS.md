# AGENTS.md

## Общ преглед на проекта

Това хранилище съдържа "AI агенти за начинаещи" - изчерпателен образователен курс, който преподава всичко необходимо за създаване на AI агенти. Курсът се състои от повече от 15 урока, покриващи основите, дизайнерски модели, рамки и производствено внедряване на AI агенти.

**Ключови технологии:**
- Python 3.12+
- Jupyter тетрадки за интерактивно обучение
- AI рамки: Semantic Kernel, AutoGen, Microsoft Agent Framework (MAF)
- Azure AI услуги: Microsoft Foundry, Azure AI Agent Service
- GitHub Models Marketplace (наличен безплатен план)

**Архитектура:**
- Структура на база уроци (директории 00-15+)
- Всеки урок съдържа: README документация, примери с код (Jupyter тетрадки) и изображения
- Поддръжка на многоезичност чрез автоматизирана система за превод
- Няколко варианта за рамки за всеки урок (Semantic Kernel, AutoGen, Azure AI Agent Service)

## Команди за настройка

### Предварителни изисквания
- Python 3.12 или по-нова версия
- GitHub акаунт (за GitHub Models - безплатен план)
- Azure абонамент (по желание, за Azure AI услуги)

### Първоначална настройка

1. **Клонирай или форкни репозитория:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # ИЛИ
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Създай и активирай виртуална Python среда:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # В Windows: venv\Scripts\activate
   ```

3. **Инсталирай зависимостите:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настрой променливи на средата:**
   ```bash
   cp .env.example .env
   # Редактирайте .env с вашите API ключове и крайни точки
   ```

### Необходими променливи на средата

За **GitHub Models (Безплатно)**:
- `GITHUB_TOKEN` - Личен достъп токен от GitHub

За **Azure AI услуги** (по желание):
- `PROJECT_ENDPOINT` - Microsoft Foundry адрес на проекта
- `AZURE_OPENAI_API_KEY` - Ключ за Azure OpenAI API
- `AZURE_OPENAI_ENDPOINT` - URL на Azure OpenAI крайна точка
- `AZURE_OPENAI_CHAT_DEPLOYMENT_NAME` - Име на внедряване на чат модел
- `AZURE_OPENAI_EMBEDDING_DEPLOYMENT_NAME` - Име на внедряване за embedding
- Допълнителна конфигурация за Azure като в `.env.example`

## Работен процес за разработка

### Стартиране на Jupyter тетрадки

Всеки урок съдържа няколко Jupyter тетрадки за различни рамки:

1. **Стартирай Jupyter:**
   ```bash
   jupyter notebook
   ```

2. **Отиди в директорията на урока** (например `01-intro-to-ai-agents/code_samples/`)

3. **Отвори и изпълнявай тетрадки:**
   - `*-semantic-kernel.ipynb` - С рамката Semantic Kernel
   - `*-autogen.ipynb` - С рамката AutoGen
   - `*-python-agent-framework.ipynb` - С Microsoft Agent Framework (Python)
   - `*-dotnet-agent-framework.ipynb` - С Microsoft Agent Framework (.NET)
   - `*-azureaiagent.ipynb` - С услугата Azure AI Agent Service

### Работа с различни рамки

**Semantic Kernel + GitHub Models:**
- Безплатен план с GitHub акаунт
- Подходящ за учене и експериментиране
- Шаблон за имена на файлове: `*-semantic-kernel*.ipynb`

**AutoGen + GitHub Models:**
- Безплатен план с GitHub акаунт
- Възможности за оркестрация на множество агенти
- Шаблон за имена на файлове: `*-autogen.ipynb`

**Microsoft Agent Framework (MAF):**
- Последна рамка от Microsoft
- Налична за Python и .NET
- Шаблон за имена на файлове: `*-agent-framework.ipynb`

**Azure AI Agent Service:**
- Изисква абонамент в Azure
- Функции готови за продукция
- Шаблон за имена на файлове: `*-azureaiagent.ipynb`

## Допълнителни бележки

### Чести проблеми

1. **Несъответствие на версията на Python:**
   - Увери се, че използваш Python 3.12+
   - Някои пакети не работят със стари версии
   - Използвай `python3 -m venv` за експлицитна версия

2. **Променливи на средата:**
   - Винаги създавай `.env` от `.env.example`
   - Не качвай `.env` файла (включен е в `.gitignore`)
   - GitHub токенът трябва да има подходящи права

3. **Конфликти в пакетите:**
   - Използвай нова виртуална среда
   - Инсталирай от `requirements.txt`, не отделни пакети
   - Някои тетрадки може да изискват допълнителни пакети, посочени в markdown клетките

4. **Azure услуги:**
   - Azure AI услугите изискват активен абонамент
   - Някои функции са регионално специфични
   - Ограниченията на безплатния план за GitHub Models са валидни

## Специфичен контекст на проекта

### Поддръжка на много езици

Това хранилище използва автоматизирана система за превод:
- Поддържат се над 50 езика
- Преводи в директории `/translations/<код-на-езика>/`
- GitHub Actions организира актуализациите на преводите
- Изходните файлове са на английски в корена на хранилището

### Структура на уроците

Всеки урок следва последователен модел:
1. Видео миниатюра с линк
2. Писмено съдържание на урока (README.md)
3. Примери с код в няколко рамки
4. Учебни цели и предварителни изисквания
5. Допълнителни учебни ресурси с линкове

### Именуване на примерните файлове с код

Формат: `<номер-на-урок>-<име-на-рамка>.ipynb`
- `04-semantic-kernel.ipynb` - Урок 4, Semantic Kernel
- `07-autogen.ipynb` - Урок 7, AutoGen
- `14-python-agent-framework.ipynb` - Урок 14, MAF Python
- `14-dotnet-agent-framework.ipynb` - Урок 14, MAF .NET

### Специални директории

- `translated_images/` - Локализирани изображения за преводи
- `images/` - Оригинални изображения за съдържание на английски
- `.devcontainer/` - Конфигурация за VS Code development контейнер
- `.github/` - GitHub Actions workflow-и и шаблони

### Зависимости

Основни пакети от `requirements.txt`:
- `autogen-agentchat`, `autogen-core`, `autogen-ext` - рамка AutoGen
- `semantic-kernel` - рамка Semantic Kernel
- `agent-framework` - Microsoft Agent Framework
- `azure-ai-inference`, `azure-ai-projects` - Azure AI услуги
- `azure-search-documents` - интеграция на Azure AI Search
- `chromadb` - векторна база данни за RAG примери
- `chainlit` - рамка за чат UI
- `browser_use` - автоматизация на браузър за агенти
- `mcp[cli]` - поддръжка на Model Context Protocol
- `mem0ai` - управление на памет за агенти
