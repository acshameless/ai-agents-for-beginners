# چک‌لیست کیفیت

قبل از تحویل یک نوت‌بوک:

- آن را حداقل یک‌بار از ابتدا تا انتها اجرا کنید (یا تا حدی که محیط اجازه می‌دهد).
- اطمینان حاصل کنید که سلول‌های ابتدای نوت‌بوک تمام حالت‌های موردنیاز را تنظیم می‌کنند؛ از حالت‌های پنهان باقی‌مانده از اجراهای قبلی خودداری کنید.
- خروجی‌ها را مرتب نگه دارید. از خروجی‌های عظیم خودداری کنید وقتی یک خلاصهٔ کوتاه کفایت می‌کند.
- ترجیحاً از جدول‌های کوچک، شاخص‌های کلیدی یا خروجی‌های چاپی کوتاه استفاده کنید.
- روایت را طوری نگه دارید که قابل مرور سریع باشد. از سرفصل‌ها و بولت‌های کوتاه استفاده کنید و از پاراگراف‌های بلند خودداری کنید.
- فقط در صورت ضرورت TODOهای مفید بگذارید و آن‌ها را واضح برچسب‌گذاری کنید.
- اگر اجرای کد ممکن نیست، ریسک را مشخص کنید و توضیح دهید چگونه می‌توان به‌صورت محلی آن را اعتبارسنجی کرد.
