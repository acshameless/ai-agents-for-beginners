# AGENTS.md

## مروری بر پروژه

این مخزن شامل «عامل‌های هوش مصنوعی برای مبتدیان» است - یک دوره آموزشی جامع که همه چیزهای لازم برای ساخت عامل‌های هوش مصنوعی را آموزش می‌دهد. این دوره شامل بیش از ۱۵ درس است که مبانی، الگوهای طراحی، چارچوب‌ها و استقرار تولیدی عامل‌های هوش مصنوعی را پوشش می‌دهد.

**تکنولوژی‌های کلیدی:**
- پایتون ۳.۱۲ و بالاتر
- دفترچه‌های Jupyter برای یادگیری تعاملی
- چارچوب‌های هوش مصنوعی: Semantic Kernel، AutoGen، چارچوب عامل مایکروسافت (MAF)
- خدمات هوش مصنوعی Azure: Microsoft Foundry، سرویس عامل هوش مصنوعی Azure
- بازار مدل‌های GitHub (لایه رایگان موجود)

**معماری:**
- ساختار بر پایه درس‌ها (دایرکتوری‌های ۰۰ تا ۱۵+)
- هر درس شامل: مستندات README، نمونه‌های کد (دفترچه‌های Jupyter) و تصاویر
- پشتیبانی چندزبانه با استفاده از سیستم ترجمه خودکار
- گزینه‌های چندگانه چارچوب در هر درس (Semantic Kernel، AutoGen، سرویس عامل هوش مصنوعی Azure)

## دستورهای راه‌اندازی

### پیش‌نیازها
- پایتون ۳.۱۲ یا بالاتر
- حساب GitHub (برای مدل‌های GitHub - لایه رایگان)
- اشتراک Azure (اختیاری، برای خدمات هوش مصنوعی Azure)

### راه‌اندازی اولیه

۱. **مخزن را کلون یا فورک کنید:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # یا
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

۲. **یک محیط مجازی پایتون بسازید و فعال کنید:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # در ویندوز: venv\Scripts\activate
   ```

۳. **وابستگی‌ها را نصب کنید:**
   ```bash
   pip install -r requirements.txt
   ```

۴. **تنظیم متغیرهای محیطی:**
   ```bash
   cp .env.example .env
   # فایل .env را با کلیدهای API و نقاط پایانی خود ویرایش کنید
   ```

### متغیرهای محیطی مورد نیاز

برای **مدل‌های GitHub (رایگان)**:
- `GITHUB_TOKEN` - توکن دسترسی شخصی از GitHub

برای **خدمات هوش مصنوعی Azure** (اختیاری):
- `PROJECT_ENDPOINT` - آدرس پروژه Microsoft Foundry
- `AZURE_OPENAI_API_KEY` - کلید API Azure OpenAI
- `AZURE_OPENAI_ENDPOINT` - آدرس نقطه پایانی Azure OpenAI
- `AZURE_OPENAI_CHAT_DEPLOYMENT_NAME` - نام استقرار مدل چت
- `AZURE_OPENAI_EMBEDDING_DEPLOYMENT_NAME` - نام استقرار مدل جاسازی
- پیکربندی‌های اضافی Azure مطابق فایل `.env.example`

## روند توسعه

### اجرای دفترچه‌های Jupyter

هر درس شامل چندین دفترچه Jupyter برای چارچوب‌های مختلف است:

۱. **اجرای Jupyter:**
   ```bash
   jupyter notebook
   ```

۲. **رفتن به دایرکتوری درس** (مثلاً `01-intro-to-ai-agents/code_samples/`)

۳. **باز کردن و اجرای دفترچه‌ها:**
   - `*-semantic-kernel.ipynb` - استفاده از چارچوب Semantic Kernel
   - `*-autogen.ipynb` - استفاده از چارچوب AutoGen
   - `*-python-agent-framework.ipynb` - استفاده از چارچوب عامل مایکروسافت (پایتون)
   - `*-dotnet-agent-framework.ipynb` - استفاده از چارچوب عامل مایکروسافت (.NET)
   - `*-azureaiagent.ipynb` - استفاده از سرویس عامل هوش مصنوعی Azure

### کار با چارچوب‌های مختلف

**Semantic Kernel + مدل‌های GitHub:**
- لایه رایگان با حساب GitHub در دسترس است
- مناسب برای یادگیری و آزمایش
- الگوی نام فایل: `*-semantic-kernel*.ipynb`

**AutoGen + مدل‌های GitHub:**
- لایه رایگان با حساب GitHub در دسترس است
- قابلیت هماهنگی چندعاملی
- الگوی نام فایل: `*-autogen.ipynb`

**چارچوب عامل مایکروسافت (MAF):**
- آخرین چارچوب از مایکروسافت
- در پایتون و .NET موجود است
- الگوی نام فایل: `*-agent-framework.ipynb`

**سرویس عامل هوش مصنوعی Azure:**
- نیاز به اشتراک Azure دارد
- قابلیت‌های آماده برای تولید
- الگوی نام فایل: `*-azureaiagent.ipynb`

۱. **تست محیط پایتون:**
   ```bash

   ```

۳. **بررسی متغیرهای محیطی:**
   ```bash
   python -c "import os; from dotenv import load_dotenv; load_dotenv(); print('✓ GITHUB_TOKEN' if os.getenv('GITHUB_TOKEN') else '✗ GITHUB_TOKEN missing')"
   ```
