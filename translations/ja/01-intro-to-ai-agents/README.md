## AIエージェントの定義と種類

### AIエージェントとは？

AIエージェントは**システム**であり、**大規模言語モデル(LLMs)** に**ツールへのアクセス**や**知識**を与えることで**行動を実行**できるようにし、その能力を拡張します。

この定義をいくつかの要素に分解してみましょう：

- **System** - エージェントを単一のコンポーネントとしてではなく、多数のコンポーネントからなるシステムとして考えることが重要です。基本的なレベルでは、AIエージェントのコンポーネントは次のとおりです：
  - **Environment** - AIエージェントが動作している定義された空間。例えば、旅行予約エージェントでは、環境はエージェントがタスクを完了するために使用する旅行予約システムである可能性があります。
  - **Sensors** - 環境には情報がありフィードバックを提供します。AIエージェントはセンサーを使用して環境の現在の状態に関するこの情報を収集および解釈します。旅行予約エージェントの例では、旅行予約システムはホテルの空室状況や航空運賃などの情報を提供できます。
  - **Actuators** - AIエージェントが環境の現在の状態を受け取ると、現在のタスクに対して環境を変化させるためにどのアクションを実行するかを決定します。旅行予約エージェントの場合、ユーザーのために利用可能な部屋を予約することが考えられます。

![AIエージェントとは何か？](../../../translated_images/ja/what-are-ai-agents.1ec8c4d548af601a.webp)

**Large Language Models** - エージェントの概念はLLMsが登場する以前から存在しました。LLMsを用いてAIエージェントを構築する利点は、人間の言語やデータを解釈する能力にあります。この能力により、LLMsは環境情報を解釈し、環境を変化させるための計画を定義できます。

**Perform Actions** - AIエージェントシステムの外部では、LLMsはユーザーのプロンプトに基づいてコンテンツや情報を生成するという動作に限定されることが多いです。AIエージェントシステム内では、LLMsはユーザーの要求を解釈し、環境で利用可能なツールを使用することでタスクを達成できます。

**Access To Tools** - LLMがアクセスできるツールは、1) それが動作している環境と、2) AIエージェントの開発者によって定義されます。旅行エージェントの例では、エージェントのツールは予約システムで利用可能な操作によって制限されるか、開発者がエージェントのツールアクセスを航空便に限定することもできます。

**Memory+Knowledge** - メモリはユーザーとエージェント間の会話の文脈では短期的である場合があります。長期的には、環境が提供する情報の外部で、AIエージェントは他のシステム、サービス、ツール、さらには他のエージェントから知識を取得することもできます。旅行エージェントの例では、この知識は顧客データベースにあるユーザーの旅行の好みに関する情報である可能性があります。

### エージェントの種類

一般的なAIエージェントの定義ができたところで、いくつかの具体的なエージェントの種類とそれらが旅行予約AIエージェントにどのように適用されるかを見てみましょう。

| **エージェントの種類**                | **説明**                                                                                                                       | **例**                                                                                                                                                                                                                   |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **単純反射エージェント**      | 事前定義されたルールに基づいて即時に行動を行います。                                                                                  | 旅行エージェントがメールの文脈を解釈し、旅行に関する苦情をカスタマーサービスに転送する。                                                                                                                          |
| **モデルベース反射エージェント** | 世界のモデルおよびそのモデルの変化に基づいて行動を行います。                                                              | 旅行エージェントは過去の価格データへのアクセスに基づき、価格変動が大きい経路を優先する。                                                                                                             |
| **目標ベースエージェント**         | 目標を解釈し、目標達成のための行動を決定することで特定の目標を達成する計画を作成します。                                  | 旅行エージェントは現在地から目的地までに必要な移動手段（車、公共交通機関、航空便など）を決定して旅程を予約する。                                                                                |
| **効用ベースエージェント**      | 好みを考慮し、トレードオフを数値的に評価して目標達成方法を決定します。                                               | 旅行エージェントは予約時に利便性とコストのトレードオフを比較して効用を最大化する。                                                                                                                                          |
| **学習エージェント**           | フィードバックに応じて行動を調整することで時間と共に改善します。                                                        | 旅行エージェントは旅行後のアンケートなどの顧客フィードバックを利用して将来の予約を調整し改善する。                                                                                                               |
| **階層型エージェント**       | 階層化されたシステムで複数のエージェントを配置し、上位エージェントがタスクを下位エージェントが完了するサブタスクに分割します。 | 旅行エージェントは旅行をキャンセルする際、タスクを（例えば特定の予約のキャンセルなどの）サブタスクに分割し、下位エージェントに完了させて上位エージェントに報告させる。                                     |
| **マルチエージェントシステム（MAS）** | エージェントは協調的または競合的に独立してタスクを完了します。                                                           | 協調：複数のエージェントがホテル、フライト、娯楽などの特定の旅行サービスを予約する。 競合：複数のエージェントが共有のホテル予約カレンダー上で顧客の予約を争う。 |

## AIエージェントをいつ使うか

前のセクションでは旅行エージェントのユースケースを使って、異なるタイプのエージェントが旅行予約のさまざまなシナリオでどのように使われるかを説明しました。このアプリケーションをコース全体で引き続き使用します。

AIエージェントが最も適しているユースケースの種類を見てみましょう：

![AIエージェントをいつ使うか？](../../../translated_images/ja/when-to-use-ai-agents.54becb3bed74a479.webp)


- **Open-Ended Problems** - LLMがタスクを完了するために必要なステップを判断することを許すケース。ワークフローに常にハードコードできるわけではないため有用です。
- **Multi-Step Processes** - 単一回の取得ではなく、エージェントが複数ターンにわたってツールや情報を使用する必要があるレベルの複雑さを要するタスク。
- **Improvement Over Time** - エージェントが環境やユーザーからのフィードバックを受け取ることで時間と共に改善し、より高い効用を提供できるタスク。

AIエージェントの使用に関するさらに多くの考慮事項は「信頼できるAIエージェントの構築」レッスンで扱います。

## エージェント的ソリューションの基本

### エージェント開発

AIエージェントシステムを設計する最初のステップは、ツール、アクション、動作を定義することです。このコースでは、エージェントを定義するために**Azure AI Agent Service**の使用に焦点を当てます。これには以下のような機能があります：

- OpenAI、Mistral、Llamaなどのオープンモデルの選択
- Tripadvisorなどのプロバイダーを通じたライセンスデータの利用
- 標準化されたOpenAPI 3.0ツールの利用

### エージェント的パターン

LLMsとのコミュニケーションはプロンプトを介して行われます。半自律的な性質のため、環境の変化の後で常に手動でLLMに再プロンプトすることは可能でも必要でもない場合があります。複数ステップにわたってよりスケーラブルにLLMにプロンプトを与えることを可能にする**エージェント的パターン**を使用します。

このコースは現在人気のあるいくつかのエージェント的パターンに分かれています。

### エージェント的フレームワーク

エージェント的フレームワークは、開発者がコードを通じてエージェント的パターンを実装できるようにします。これらのフレームワークは、エージェントの協調を向上させるためのテンプレート、プラグイン、ツールを提供します。これらの利点により、AIエージェントシステムの可観測性やトラブルシューティングが向上します。

このコースでは、研究駆動の AutoGen フレームワークと、実運用向けの Semantic Kernel の Agent フレームワークを探ります。
