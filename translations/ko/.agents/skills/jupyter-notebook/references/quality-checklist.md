# 품질 체크리스트

Before delivering a notebook:

- 최소 한 번은 위에서 아래로 실행하세요(또는 환경이 허용하는 만큼 실행하세요).
- 초기 셀들이 필요한 모든 상태를 설정하도록 하세요; 이전 실행의 숨겨진 상태를 피하세요.
- 출력은 깔끔하게 유지하세요. 짧은 요약으로 충분하면 거대한 출력을 피하세요.
- 작은 테이블, 핵심 지표, 또는 짧은 출력물을 선호하세요.
- 내용은 훑어보기 쉽게 유지하세요. 제목과 짧은 글머리표를 사용하고 긴 문단은 피하세요.
- 필요한 경우에만 유용한 TODO를 남기고, 명확하게 표시하세요.
- 실행이 불가능한 경우, 위험을 명시하고 로컬에서 어떻게 검증할지 설명하세요.
