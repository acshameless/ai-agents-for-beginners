# စမ်းသပ်မှု ပုံစံများ

ဤ ဖွဲ့စည်းမှုကို ရှာဖွေစမ်းသပ်မှုများနှင့် စမ်းသပ်ရေးအလုပ်များအတွက် အသုံးပြုပါ:

- ခေါင်းစဉ်နှင့် ရည်ရွယ်ချက်: မေးခွန်းကို ဖော်ပြပြီး အောင်မြင်မှု သတ်မှတ်ချက်များကို ဖော်ပြပါ။
- ပြင်ဆင်ခြင်းနှင့် ထပ်မံ ထုတ်လုပ်နိုင်ခြင်း: လိုအပ်သလောက်သာ import ပြုလုပ်ပါ၊ အစပိုင်းတွင် seed သတ်မှတ်ပါ၊ configuration များကို တစ်ခုတည်းသော short cell ထဲတွင် ထားပါ။
- အစီအစဉ်: hypotheses, sweeps, နှင့် metrics များကို ကုဒ် run မလုပ်မီ စာရင်းပြုစုပါ။
- အနည်းဆုံး အခြေခံ: အနည်းဆုံး runnable ဖြစ်နိုင်သည့် ဥပမာငယ်ဖြင့် စတင်ပြီး ရှုပ်ထွေးမှု ထည့်မည့် မတိုင်ခင် အစမှ အဆုံး အလုပ်လုပ်သည်ကို အတည်ပြုပါ။
- ရလဒ်များနှင့် မှတ်စုများ: ဆက်စပ်ကုဒ်အနီး markdown တွင် ရလာသော ရှာဖွေတွေ့ရှိချက်များကို အကျဉ်းချုပ်ရေးပြီး အဓိက metrics များကို သေးငယ်သော dictionary သို့မဟုတ် ဇယားကဲ့သို့ ဖွဲ့စည်းမှုတစ်ခုတွင် မှတ်တမ်းတင်ပါ။
- နောက်တစ်ဆင့်များ: ဆက်လုပ်မလား၊ pivot လုပ်မလား သို့မဟုတ် ရပ်တန့်မလား ဆုံးဖြတ်ပြီး လိုက်လံဆောင်ရွက်နိုင်သည့် နောက်ဆက်တွဲ အကြံများကို တိုတောင်းသော bullet များအနက် မှတ်တမ်းတင်ပါ။
