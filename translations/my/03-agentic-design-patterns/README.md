# AI Agentic ဒီဇိုင်း နိယာမများ

## နိဒါန်း

AI Agentic စနစ်များကို တည်ဆောက်ရန် ဆင်ဆိုင်နိုင်သည့် နည်းလမ်းများ များပြားသည်။ Generative AI ဒီဇိုင်းတွင် မရိုးရှင်းသော အမြင်များကို အင်ဂျင်နီယာများအတွက် စတင်ရန် အခက်အခဲ ဖြစ်စေနိုင်သောကြောင့်၊ သုံးစွဲသူအခြေစိုက် agentic စနစ်များကို ဖန်တီးနိုင်ရန် လူမားပြု UX ဒီဇိုင်း နိယာမများကို ကျွန်တော်တို့ ဖန်တီးထားပါသည်။ ဤဒီဇိုင်းနိယာမများမှာ တိကျသည့် အင်ဂျင်နီယာဘဏ်ပုံသဏ္ဍာန်မဟုတ်ဘဲ agent အတွေ့အကြုံများကို သတ်မှတ်၍ တည်ဆောက်နေသော အသင်းများအတွက် စတင်ရန် အထောက်အကူဖြစ်စေမည့် အချက်များဖြစ်သည်။

အထွေထွေအားဖြင့်၊ agent များသည် -

- လူသားများ၏ စွမ်းရည်များကို ကျယ်ပြန့်စေခြင်းနှင့် အရွယ်ချဲ့စေခြင်း (စိတ်ကူးရှာဖွေခြင်း၊ ပြဿနာ ဖြေရှင်းခြင်း၊ အလိုအလျောက် ဆောင်ရွက်ခြင်း စတဲ့ အရာများ)
- သိမှတ်မှု အားကောင်းစေခြင်း (ကျွန်ုပ်တို့ကို သိပညာ အကြောင်းရင်းများ၊ ဘာသာပြန်ခြင်း စသည့် အရာများတွင် အမြန်သိရှိစေခြင်း)
- တွဲဖက်လုပ်ဆောင်ခြင်းကို အလားအလာနှင့် ကိုက်ညီစေရန် အထောက်အကူ ပြုခြင်း
- လူအဖြစ် ပိုမိုကောင်းမွန်စေရန် ကူညီခြင်း (ဥပမာ - အသက်မွေးဝမ်းကြောင်း ကိုယ်ပိုင် ကိုင်တွယ်မှု/တာဝန်ကျေမှု၊ စိတ်သိမြင် စီမံရန်ကျွမ်းကျင်စေခြင်း၊ တည်ငြိမ်မှု တိုးတက်စေခြင်း စသည်)

## ဤသင်ခန်းတွင် လေ့လာရမည့် အရာများ

- Agentic ဒီဇိုင်း နိယာမများ란 무엇인가
- ဤဒီဇိုင်း နိယာမများကို အကောင်အထည်ဖော်စဉ် လိုက်နာရမည့် လမ်းညွှန်များ
- ဒီဇိုင်း နိယာမများကို အသုံးပြုပြီး ဥပမာများ

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤသင်ခန်းကို ပြီးမြောက်ပြီးနောက်၊ သင်သည် အောက်ပါများကို ဆောင်ရွက်နိုင်မည်ဖြစ်သည်။

1. Agentic ဒီဇိုင်း နိယာမများဘာလဲဆိုတာ ရှင်းပြနိုင်ရန်
2. Agentic ဒီဇိုင်း နိယာမများ အသုံးပြုရန် လမ်းညွှန်ချက်များကို ရှင်းပြနိုင်ရန်
3. Agentic ဒီဇိုင်း နိယာမများကို အသုံးပြု၍ agent တစ်ခု တည်ဆောက်နည်းကို နားလည်နိုင်ရန်

## Agentic ဒီဇိုင်း နိယာမများ

![Agentic ဒီဇိုင်း နိယာမများ](../../../translated_images/my/agentic-design-principles.1cfdf8b6d3cc73c2.webp)

### Agent (Space)

ဤသည်မှာ agent が လှုပ်ရှားသော ပတ်ဝန်းကျင်ဖြစ်သည်။ ဤနိယာမများက physical နှင့် digital ကမ္ဘာများတွင် ပတ်သက်ဆက်ဆံရာတွင် agent များကို မည်သို့ ဒီဇိုင်းလုပ်ရမည်ကို မူတည်စေသည်။

- **ချိတ်ဆက်ပေးခြင်း၊ ဖျက်စီးပစ်ခြင်းမဟုတ်** – လူများကို အခြားလူများ၊ ဖြစ်ရပ်များနှင့် လုပ်ဆောင်နိုင်သည့် အချက်အလက်များထံ ချိတ်ဆက်ပေးကာ ပူးပေါင်းဆောင်ရွက်ခြင်းနှင့် ချိတ်ဆက်မှုကို အားပေးရန်။
- Agent များသည် ဖြစ်ရပ်များ၊ သိပညာနှင့် လူများကို ချိတ်ဆက်ပေးသည်။
- Agent များက လူများကို နီးကပ်လာစေသည်။ လူများကို အစားထိုးရန် သို့မဟုတ် ချစ်စဖျောက်ရန် မဒီဇိုင်းထားထားပါ။
- **လွယ်ကူစွာ ဝင်ရောက်အသုံးပြုနိုင်သော်လည်း မကြာခဏ မမြင်မကွင်း ဖြစ်နိုင်ခြင်း** – agent သည် အများအားဖြင့် နောက်ခံတွင် လည်ပတ်ပြီး သက်ဆိုင်ရာနှင့် သင့်တော်သောအခါသာ သတိပေးမှုများပေးသည်။
  - Agent သည် အတည်ပြုထားသည့် အသုံးပြုသူများအတွက် မည်သည့် စက်ပစ္စည်း သို့မဟုတ် ပလက်ဖောင်းပေါ်တွင်မဆို လွယ်ကူစွာ ရှာဖွေ တွေ့ရှိနိုင်ပြီး ဝင်ရောက် အသုံးပြုနိုင်ရမည်။
  - Agent သည် မော်ဒယ်စုံဖြစ်သည့် input/output များ (အသံ၊ အသံစကား၊ စာသား စသည်) ကို ပံ့ပိုးပေးရမည်။
  - Agent သည် မျက်နှာပြင်ရှေ့နှင့် နောက်ခံအကြား၊ ကြိုတင်လုပ်ဆောင်မှုနှင့် တုံ့ပြန်မှု အခြေအနေများအကြား ဆက်လက်ပြောင်းလဲနိုင်ရမည်၊ အသုံးပြုသူလိုအပ်ချက်များကို ခံစားနိုင်မှုအပေါ် မူတည်ပြီး။
  - Agent သည် မမြင်နိုင်သည့် ပုံစံဖြင့် လည်ပတ်နိုင်သော်လည်း ၎င်း၏ နောက်ခံလုပ်ငန်းလမ်းကြောင်းနှင့် အခြား Agent များနှင့် ပူးပေါင်းဆောင်ရွက်မှုသည် အသုံးပြုသူအနေဖြင့် ထင်ရှားပြီး ထိန်းချုပ်နိုင်ရမည်။

### Agent (Time)

ဤသည်မှာ agent が အချိန်အရ လည်ပတ်သည့် နည်းကို ဆိုလိုသည်။ ဤနိယာမများက အတိတ်၊ ယခုနှင့် အနာဂတ်အတွင်း ဆက်ဆံရာတွင် agent များကို မည်သို့ ဒီဇိုင်းလုပ်ရမည်ကို လမ်းညွှန်ပေးသည်။

- **အတိတ်**: အခြေအနေ နှင့် ပတ်ဝန်းကျင်ပါသည့် မှတ်တမ်းများကို ထပ်လေ့လာခြင်း။
  - Agent သည် အဖြစ်အပျက်၊ လူများ သို့မဟုတ် အခြေအနေများသာမက ပိုမိုထူထပ်သော သမိုင်းဆိုင်ရာဒေတာ ချီးမြှင့် အချက်အလက်များအပေါ် အခြေခံ၍ ပိုသက်သာသော ရလဒ်များ ပေးနိုင်စေသည်။
  - Agent သည် အတိတ် ဖြစ်ရပ်များမှ ချိတ်ဆက်ချက်များ ဖန်တီးပြီး မှတ်ဉာဏ်ကို တက်ကြွစွာ ပြန်လည်စဉ်းစားကာ လက်ရှိ အခြေအနေများနှင့် ဆက်သွယ်ပါသည်။
- **ယခု**: ဆောင်ကြိုးပို၍ အသိပေးပေးခြင်းထက် မျှော်လင့်ချက်ပေးခြင်း။
  - Agent သည် လူများနှင့် ဆက်သွယ်ရာတွင် ကျယ်ပြန့်ပြီး စုံလင်သော နည်းလမ်းကို ကိုင်ဆောင်ထားသည်။ ဖြစ်ရပ်တစ်ခု ဖြစ်ပျက်သောအခါ၊ Agent သည် တည်ငြိမ်သည့် သတိပေးချက် သာမက ဂရုစိုက်စေသော နည်းလမ်းများကို အထူးပြု သို့မဟုတ် dynamic ဖြစ်စေရန် အချက်အလက်များ ဖန်တီးနိုင်သည်။
  - Agent သည် ပတ်ဝန်းကျင်ဆိုင်ရာ အခြေအနေ၊ လူမှုနှင့် ယဉ်ကျေးမှု ပြောင်းလဲမှုများနှင့် အသုံးပြုသူရည်ရွယ်ချက်အပေါ် အခြေခံ၍ အချက်အလက်များ ပေးပို့သည်။
  - Agent နှင့်의 ဆက်သွယ်မှုသည် တဖြည်းဖြည်း ဖြစ်စေ၍ ထာလေရှည်အတွင်း အသုံးပြုသူအား အင်အားဖြည့်ပေးမည့် အဆင့်မြှင့်သွားနိုင်သည်။
- **အနာဂတ်**: ကိုက်ညီ၍ တိုးတက်လာခြင်း။
  - Agent သည် မျိုးစုံ စက်ပစ္စည်းများ၊ ပလက်ဖောင်းများနှင့် မော်ဒယ်များအပေါ် ကိုက်ညီသွားနိုင်သည်။
  - Agent သည် အသုံးပြုသူ အပြုအမူ၊ အသုံးပြုနိုင်မှု လိုအပ်ချက်များကို ကိုက်ညီပြီး လွယ်လင့်ချိန် သတ်မှတ်နိုင်စေရန် ချိန်ညှိနိုင်ရမည်။
  - Agent သည် အသုံးပြုသူ၏ ဆက်တိုက်ဆက်လက်ဆက်ဆံမှုများမှတဆင့် ပုံဖော်ကာ တိုးတက်လာသည်။

### Agent (Core)

ဤများသည် agent ၏ မူလဒီဇိုင်း အဓိက ဗဟိုပိုင်း အစိတ်အပိုင်းများဖြစ်သည်။

- **မရှင်းလင်းမှုကို လက်ခံကာ ယုံကြည်မှု တည်ဆောက်ပါ**။
  - Agent ၏ မရှင်းလင်းမှု အသ一定 ရှိမည်ဟု မျှော်လင့်နိုင်သည်။ မရှင်းလင်းမှုသည် agent ဒီဇိုင်း၏ အဓိက အစိတ်အပိုင်းတစ်ခုဖြစ်သည်။
  - ယုံကြည်မှုနှင့် သတ္တိကျင့မှုသည် Agent ဒီဇိုင်း၏ မူလ အလွှာများဖြစ်သည်။
  - လူများဟာ Agent ကို မည်တော့မှာ On/Off ထားမည်ကို ထိန်းချုပ်နိုင်ပြီး Agent ၏ အခြေအနေကို မည်သည့်အချိန်တွင်မဆို ပြသထားရမည်။

## ဤနိယာမများကို အကောင်အထည်ဖော်ရန် လမ်းညွှန်ချက်များ

အထက်ဖော်ပြသော ဒီဇိုင်းနိယာမများကို အသုံးပြုနေစဉ်တွင် အောက်ပါ လမ်းညွှန်ချက်များကို အသုံးပြုပါ။

1. **ထင်ရှားမှု (Transparency)**: AI ပါဝင်သည်ဟု အသုံးပြုသူကို အသိပေးပါ၊ ၎င်းသည် မည်သို့ လည်ပတ်သည်ကို (အတိတ် လုပ်ဆောင်ချက်များအပါအဝင်) နှင့် ပြန်လည်တုံ့ပြန်မှုပေးရန် နှင့် စနစ်ကို ပြင်ဆင်ရန် မည်သို့ လုပ်ရမည်ကို အချက်အလက်ပေးပါ။
2. **ထိန်းချုပ်မှု (Control)**: အသုံးပြုသူအား စိတ်ကြိုက် ပြင်ဆင်ခြင်း၊ Preferences သတ်မှတ်ခြင်းနှင့် ကိုယ်ပိုင်ပြုလုပ်နိုင်စေရန်၊ စနစ်နှင့် ၎င်း၏ Attribute များအပေါ် ထိန်းချုပ်ခွင့်ပေးပါ (မေ့ဖို့ စွမ်းရည်ပါဝင်ခြင်းအပါအဝင်)။
3. **တည်တူညီမျှမှု (Consistency)**: စက်ပစ္စည်းနှင့် အချက်အပြားများအနှံ့ မျိုးစုံ modality အတွေ့အကြုံများကို တည်တူစေဖို့ ကြိုးပမ်းပါ။ မဖြစ်မနေလိုက်ဖက်သည့် UI/UX အစိတ်အပိုင်းများ (ဥပမာ - အသံဆက်သွယ်မှုအတွက် မိုက်ခရိုဖုန်း အိုင်ကွန်) ကို အသုံးပြုပြီး ဖောက်သည်၏ စိတ်ပိုင်းဆိုင်ရာ ပင်ပန်းမှုကို များစွာလျော့ပါးစေပါ (ဥပမာ - အတိုချုံးဖြေတုံ့ပြန်ချက်များ၊ မြင်နိုင်သည့် ကူညီပစ္စည်းများနှင့် 'Learn More' အကြောင်းအရာများ)။

## ဤနိယာမများနှင့် လမ်းညွှန်ချက်များကို အသုံးပြု၍ တစ်ဦး Travel Agent ကို မည်သို့ ဒီဇိုင်းလုပ်မည်ဆိုသည်ကို စဉ်းစားပါ

သင်သည် Travel Agent တစ်ခုကို ဒီဇိုင်းလုပ်နေသည်ဟု သုံးစားပါ၊ ဒီအတိုင်း ဒီဇိုင်းနိယာမများနှင့် လမ်းညွှန်ချက်များကို မည်သို့ အသုံးချနိုင်သည်ကို စဉ်းစားကြည့်ပါ။

1. **Transparency (ထင်ရှားမှု)** – အသုံးပြုသူသို့ Travel Agent သည် AI ဖြင့် စွမ်းဆောင်နိုင်သော Agent ဖြစ်ကြောင်း အသိပေးပါ။ စတင်အသုံးပြုရန် အချက်အလမ်းအနည်းငယ် ပေးပါ (ဥပမာ - "မင်္ဂလာပါ" စာတန်း၊ နမူနာ prompt များ)။ ထို့ကို ထုတ်ကုန်စာမျက်နှာတွင် ရှင်းလင်းစွာ မှတ်သားပါ။ အသုံးပြုသူက မပြောခဲ့ဖူးသော prompt များစာရင်းကို ပြပါ။ အကြိုက်မှန်းသတ်ပြရန် (thumbs up/down၊ Send Feedback ခလုတ် စသည်) မည်သို့ ပြောရမည်ကို ရှင်းလင်းပြပါ။ Agent ၏ အသုံးပြုမှု သို့မဟုတ် ခေါင်းစဉ် ကန့်သတ်ချက်များ ရှိပါက ထပ်မံ ရှင်းပြပါ။
2. **Control (ထိန်းချုပ်မှု)** – Agent ကို တည်ဆောက်ပြီးနောက် System Prompt ကဲ့သို့သော အရာများဖြင့် အသုံးပြုသူ ပြင်ဆင်နိုင်မည့် နည်းလမ်းများ ရှင်းလင်းစေရန် အာမခံပါ။ အသုံးပြုသူအား Agent ၏ စာရေးပုံစံ၊ အကြောင်းအရာ ရှင်းလင်းမှု အလျော့ကို ရွေးချယ်နိုင်စေပါ။ Agent မပြောသင့်သော အကြောင်းအရာများကို သတ်မှတ်လိုက်ပါ။ အသုံးပြုသူအား ဆက်စပ် ဖိုင်များ သို့မဟုတ် ဒေတာများ၊ prompt များနှင့် မျက်နှာတိုက် စကားသင်ခန်းများကို ကြည့်ရှု၍ ဖျက်ပစ်နိုင်စေပါ။
3. **Consistency (တည်တူညီမျှမှု)** – Share Prompt, ဖိုင် သို့မဟုတ် ဓာတ်ပုံ တင်ရန်နှင့် ပုဂ္ဂိုလ်တစ်ဦး သို့မဟုတ် အရာတစ်ခုကို တက်ပြီတင်ရန် အိုင်ကွန်များကို မူရင်း နံပါတ်နှင့် မှတ်သားနိုင်စေရန် သေချာစေပါ။ ဖိုင် ထည့်သွင်းသို့ မျှဝေမှုအတွက် paperclip အိုင်ကွန်ကို အသုံးပြု၍ ဂရပ်ဖစ် တင်ရန် image အိုင်ကွန်ကို အသုံးပြုပါ။
