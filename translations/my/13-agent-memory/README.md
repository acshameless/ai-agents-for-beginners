# AI အေဂျင့်များအတွက် မှတ်ဉာဏ် 
[![အေဂျင့် မှတ်ဉာဏ်](../../../translated_images/my/lesson-13-thumbnail.959e3bc52d210c64.webp)](https://youtu.be/QrYbHesIxpw?si=qNYW6PL3fb3lTPMk)

AI အေဂျင့်များ ဖန်တီးရာတွင် ထူးခြားသည့် အကျိုးကျေးဇူးနှစ်ချက်ကို များအားဖြင့် ဆွေးနွေးကြသည် — အလုပ်များကို ပြီးရမှတ်စေသည့် tools များကို ခေါ်နိုင်ခြင်းနှင့် အချိန်အတိုင်းအတာဖြင့် တိုးတက်လာနိုင်ခြင်း။ မှတ်ဉာဏ်သည် ကိုယ်တိုင်တိုးတက်လာနိုင်သော agent တစ်ခု ဖန်တီးရာတွင် အခြေခံဖြစ်ပြီး အသုံးပြုသူများအတွက် ပိုမိုကောင်းမွန်သော အတွေ့အကြုံများ ဖန်တီးနိုင်စေသည်။

ဤသင်ခန်းစာတွင် ကျွန်ုပ်တို့သည် AI အေဂျင့်များအတွက် မှတ်ဉာဏ် ဆိုသည်မှာ အဘယ်၍ ဖြစ်ပြီး ဤကို မည်သို့စီမံနှင့် အသုံးချနိုင်ကြောင်း ကြည့်ပါမည်။

## နိဒါန်း

ဤသင်ခန်းစာတွင် ဖော်ပြမည့် အချက်များမှာ -

• **AI အေဂျင့် မှတ်ဉာဏ် ကို နားလည်ခြင်း**: မှတ်ဉာဏ်ဆိုတာဘာလဲ၊ အဘယ်ကြောင့် အရေးကြီးတာလဲ။

• **မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်ခြင်းနှင့် သိမ်းဆည်းခြင်း**: အတိုချုံးနှင့် ရေရှည်မှတ်ဉာဏ်တို့ကို ရည်ရွယ်၍ AI အေဂျင့်များတွင် မှတ်ဉာဏ်စွမ်းရည် ထည့်သွင်းပေးခြင်း၏ လက်တွေ့နည်းလမ်းများ။

• **AI အေဂျင့်များကို ကိုယ်တိုင်တိုးတက်လာစေခြင်း**: အရင်က ဆက်ဆံမှုများမှ သင်ယူ၍ အချိန်အတော်ကြာ တိုးတက်လာစေဖို့ မှတ်ဉာဏ်ဘယ်လို အထောက်အကူပြုသလဲ။

## ရနိုင်သော အကောင်အထည်ဖော်ချက်များ

ဤသင်ခန်းစာတွင် နောက်လိုက်နှစ်ခုသော notebook သင်ခန်းစာများ ပါဝင်သည်။

• **[13-agent-memory.ipynb](./13-agent-memory.ipynb)**: Mem0 နှင့် Azure AI Search ကို Semantic Kernel framework ဖြင့် အသုံးပြုကာ မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်သည်။

• **[13-agent-memory-cognee.ipynb](./13-agent-memory-cognee.ipynb)**: Cognee ကို အသုံးပြု၍ ဖွဲ့စည်းတည်ဆောက်ထားသော မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်ပြီး embeddings အား အထောက်အထားပြု၍ အသိပညာဂရပ် ဖန်တီးပေးခြင်း၊ ဂရပ်ကို မြင်ကွင်းပေးခြင်းနှင့် ထောက်ယူမှုများကို ထိထိရောက်ရောက် လုပ်ဆောင်သည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤသင်ခန်းစာပြီးဆုံးသည့်နောက် သင်သည် သိထားမည်မှာ -

• **AI အေဂျင့်မှတ်ဉာဏ်၏ မတူကွဲပြားသော အမျိုးအစားများကို ခွဲခြားနားလည်နိုင်ခြင်း** — အလုပ်လုပ်နေစဉ်မှတ်ဉာဏ် (working), အတိုချုံးမှတ်ဉာဏ် (short-term), ရေရှည်မှတ်ဉာဏ် (long-term) အပါအဝင် persona နှင့် episodic memory ကဲ့သို့ အထူးပြုအမျိုးအစားများကိုပါ။

• **Semantic Kernel framework ကို အသုံးပြု၍ AI အေဂျင့်များအတွက် အတိုချုံးနှင့် ရေရှည်မှတ်ဉာဏ်တပ်ဆင်ခြင်းနှင့် စီမံခန့်ခွဲခြင်း** ကို Mem0, Cognee, Whiteboard memory ကဲ့သို့သော tools များနှင့် Azure AI Search ကို ဆက်စပ်၍ လက်တွေ့ အကောင်အထည်ဖော်နိုင်ခြင်း။

• **ကိုယ်တိုင်တိုးတက်လာနိုင်သော AI အေဂျင့်များ၏ အခြေခံသဘောတရားများကို နားလည်ခြင်း** ၊ သံသရာကောင်းမွန်သော မှတ်ဉာဏ် စီမံခန့်ခွဲမှုစနစ်များက ဆက်လက်လေ့လာခြင်းနှင့် လိုက်လျောညီမှုများကို အထောက်အကူပြုကြောင်း ကို သိရှိနိုင်ခြင်း။

## AI အေဂျင့်မှတ်ဉာဏ်ကို နားလည်ခြင်း

အက်ဦးတွင်၊ **AI အေဂျင့်များအတွက် မှတ်ဉာဏ် ဆိုသည်မှာ ၎င်းတို့အား သတင်းအချက်အလက်ကို သိမ်းဆည်းထားနိုင်ခြင်းနှင့် ပြန်လည်ခေါ်ယူနိုင်စေရန် ရည်ရွယ်သည့် ကိရိယာများဖြစ်သည်**။ ၎င်းသတင်းအချက်အလက်များသည် စကားပြောဆိုမှုအကြောင်းအရာအချက်အလက်များ၊ အသုံးပြုသူနှစ်သက်ရာများ၊ အလုပ်ဆောင်ချက်အရင်းအမြစ်များ သို့မဟုတ် သင်ယူထားသော အဆင့်လျှောက်များဖြစ်နိုင်သည်။

မှတ်ဉာဏ်မရှိပါက AI လျာထားမှုများသည် ပုံမှန်အားဖြင့် stateless ဖြစ်ပြီး တစ်ခုချင်းစီသော ဆက်ဆံမှုသည် အစပျိုးနေစဉ်မှ စတင်သည်။ ၎င်းကြောင့် agent က မတိုင်ခင် context သို့မဟုတ် အသုံးပြုသူနှစ်သက်ချက်များကို "မေ့နေသည်" ကဲ့သို့ အလှည့်အပြောင်းများများ ဖြစ်ပေါ်စေပြီး အသုံးပြုသူအတွက် ထပ်တလဲလဲနှင့် စိတ်မရှည်နိုင်စေသော အတွေ့အကြုံကို ပေးစေသည်။

### မွတ်ဉာဏ်က အဘယ်ကြောင့် အရေးကြီးသနည်း?

agent တစ်ယောက်၏ တာဝန်ခံ ဉာဏ်ရည်သည် အရင်က သိထားသည့် အချက်အလက်များကို ပြန်လည် သုံးနိုင်စွမ်းနှင့် သက်ရောက်မှုရှိစွာ ချိတ်ဆက်နိုင်စွမ်းနှင့် ဝိုင်လင်။ မှတ်ဉာဏ်ကြောင့် agent များသည် အောက်ပါအတိုင်း ဖြစ်လာနိုင်သည်။

• **ပြန်လှုံ့ဆော်နိုင်ခြင်း (Reflective)**: အရင်လုပ်ဆောင်ချက်များနှင့် ရလဒ်များမှ သင်ယူခြင်း။

• **ချိတ်ဆက်ဆက်ဆံနိုင်ခြင်း (Interactive)**: ဆက်လက်ဆက်ဆံနေသော စကားဝိုင်းတွင် context ကို ထိမ်းသိမ်းထားနိုင်ခြင်း။

• **ရင်ဆိုင်တုံ့ပြန်နိုင်ခြင်းနှင့် ကြိုတင်လုပ်ဆောင်နိုင်ခြင်း (Proactive and Reactive)**: အတိတ်ဒေတာအပေါ်အခြေခံ၍ လိုအပ်ချက်များကို ခန့်မှန်းခြင်း သို့မဟုတ် သင့်လျော်စွာ တုံ့ပြန်နိုင်ခြင်း။

• **အလိုအလျောက် စွမ်းဆောင်နိုင်ခြင်း (Autonomous)**: သိမ်းဆည်းထားသည့် ဗဟုသုတကို အချို့ဆောင်ရွက်မှုများတွင် အသုံးချ၍ ပို၍ လွတ်လပ်စွာ လည်ပတ်နိုင်ခြင်း။

မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်ရသည့် ရည်ရွယ်ချက်မှာ agent များကို ပိုမို **ယုံကြည်ရစေပြီး စွမ်းရည်ရှိစေခြင်း** ဖြစ်သည်။

### မှတ်ဉာဏ်အမျိုးအစားများ

#### အလုပ်လုပ်နေစဉ်မှတ်ဉာဏ် (Working Memory)

ဤကို တစ်စိတ်တစ်ပိုင်း သွက်လက်လှုပ်ရှားစဉ်၌ agent အနေဖြင့် အသုံးပြုသည့် စာရွက်ကုံးတစ်စိတ်အဖြစ် တွေးပါ။ နောက်တစ်ခြား အဆင့်တွက်ချက်ရန် လတ်တလောလိုအပ်သည့် သတင်းအချက်အလက်များကို သိမ်းဆည်းထားသည်။

AI အေဂျင့်များအတွက် အလုပ်လုပ်နေစဉ်မှတ်ဉာဏ်သည် မျက်နှာသာပြောဆိုမှု၏ အဓိကသက်ဆိုင်ရာ အချက်အလက်များကို ဖမ်းယူလေ့ရှိသည်၊ စကားပြောအစုံအလင် သို့မဟုတ် အတွဲအစုံဟာ အရှည်မျိုးမဖြစ်ခဲ့ သိမ်းဆည်းမရတဲ့အခါတွင်ပါ။ ၎င်းသည် လိုအပ်ချက်များ၊ အကြံပြုချက်များ၊ ဆုံးဖြတ်ချက်များနှင့် လုပ်ဆောင်ချက်များကဲ့သို့သော အဓိက အချက်အလက်များကို ထုတ်ချထားခြင်းအပေါ် ကောင်းစွာ ဦးတည်သည်။

**အလုပ်လုပ်နေစဉ်မှတ်ဉာဏ် ဥပမာ**

ခရီးစဉ်ဘွတ်ကင်ချရန် agent တစ်ယောက်တွင် အလုပ်လုပ်နေစဉ်မှတ်ဉာဏ်သည် အသုံးပြုသူ၏ လက်ရှိတောင်းဆိုချက် "Paris သို့ ခရီးစဉ်စာရင်း မှာချင်ပါတယ်" ဆိုသော အချက်ကို ဖမ်းယူထားပြီး လက်ရှိ ဆက်ဆံမှုကို ဦးတည်စေသည်။

#### အတိုချုံးမှတ်ဉာဏ် (Short Term Memory)

ဤအမျိုးအစားမှတ်ဉာဏ်သည် တစ်စကားဝိုင်း သို့မဟုတ် session တစ်ခု၏ ရာသီအတွင်း သာ သတင်းအချက်အလက်ကို ထိန်းသိမ်းထားသည်။ ၎င်းသည် လက်ရှိ chat ၏ context ဖြစ်ပြီး agent ကို စကားအတွင်း ယခင် turn များကို ပြန်လှည့်မြင်နိုင်စေသည်။

**အတိုချုံးမှတ်ဉာဏ် ဥပမာ**

အသုံးပြုသူက "Paris သို့ စက်ဘီးလေယာဉ်ဘတ်(ခ) ငွေ ဘယ်လောက်ကျမလဲ?" ဟုပြီးနောက် "အဲဒီမှာနေထိုင်ခင်းကရော?" ဟုပြောပါက အတိုချုံးမှတ်ဉာဏ်ကြောင့် agent သည် "there" ဟူသောအရာသည် အဆိုပါ စကားဝိုင်းအတွင်း "Paris" ကို ရည်ညွှန်းသည်ဟု သိနိုင်သည်။

#### ရေရှည်မှတ်ဉာဏ် (Long Term Memory)

ဤသည်မှာ စကားဝိုင်းများ သို့မဟုတ် session များ အစဉ်အမြဲကို ကျော်လွန်၍ ရပ်တည်နေသော သတင်းအချက်အလက်ဖြစ်သည်။ ၎င်းက အသုံးပြုသူနှစ်သက်ချက်များ၊ သမိုင်းကြောင်းဆက်ဆံမှုများ သို့မဟုတ် ရေရှည်ကျော်လွန်သိရှိနိုင်သည့် အထွေထွေ ဗဟုသုတများကို မှတ်ထားနိုင်သည်။ ဤသည်မှာ ပုဂ္ဂိုလ်ရေးပြုစိတ်နှင့် ကိုက်ညီမှုအတွက် အဓိက အရေးပါတဲ့အရာဖြစ်သည်။

**ရေရှည်မှတ်ဉာဏ် ဥပမာ**

ရေရှည်မှတ်ဉာဏ်တွင် "Ben သည် ခဲယန်းစီးခြင်းနှင့် ပြင်ပလှည်းလုပ်ငန်းများကို နှစ်သက်သည်၊ တောင်တန်းမြင်ကွင်း付き ကော်ဖီကို ကြိုက်သည်၊ နှင့် အရင်ထိခိုက်မှုကြောင့် အဆင့်မြင့် ski ဖြတ်လမ်းများ မှကာကွယ်လိုသည်" လိုသော အချက်များကို သိမ်းဆည်းနိုင်သည်။ အဆိုပါ အချက်များသည် အနာဂတ်ခရီးစဉ်စီစဉ်မှုများတွင် အကြံပြုချက်များကို ပုဂ္ဂိုလ်ရေးတက်ကျစေသည်။

#### Persona မှတ်ဉာဏ်

ဤအထူးပြုမှတ်ဉာဏ်အမျိုးအစားသည် agent တစ်ယောက်အတွက် တည်ငြိမ်သော "persona" သို့မဟုတ် "ကိုယ်ရည်ကိုယ်သွေး" တစ်ခု ဖွံ့ဖြိုးစေသည်။ ၎င်းက agent ကို ၎င်း၏ ကိုယ်ပိုင်သတင်းအချက်အလက် သို့မဟုတ် မည်သည့် အခန်းကဏ္ဍကို ဆောင်ရွက်သင့်ကြောင်း မှတ်ထားစေပြီး ဆက်ဆံရေးကို ပိုမိုစီးဆင်းလွယ်စေသည်။

**Persona မှတ်ဉာဏ် ဥပမာ**
ခရီးစဉ် agent ကို "ကျွမ်းကျင်သော ski စီစဉ်သူ" အဖြစ် ဒီဇိုင်းတည်ထောင်ထားပါက persona မှတ်ဉာဏ်သည် ၎င်း၏ အခန်းကဏ္ဍကို အတည်ပြုကာ ၎င်း၏ ဖြေကြားချက်များတွင် ကျွမ်းကျင်သူ၏ အသံနှင့် သိရှိမှုကို ထိန်းသိမ်းစေမည်။

#### Workflow/Episodic မှတ်ဉာဏ်

ဤမှတ်ဉာဏ်သည် agent တစ်ယောက်က ကြုံတွေ့သော ပြဿနာရှုပ်ထွေးသော လုပ်ငန်းစဉ်တစ်ခုအတွင်း ခရီးစဉ်ဆက်လက်သို့ ရောက်ရှိသော အဆင့်များ (အောင်မြင်မှုများနှင့် တက်လျှောက်မှုများ) ကို သိမ်းဆည်းထားသည်။ ၎င်းသည် အမှတ်ရထားသည့် "episodes" များသလို ဖြစ်ပြီး အတိတ်အတွေ့အကြုံများမှ သင်ယူရန် အသုံးပြုသည်။

**Episodic မှတ်ဉာဏ် ဥပမာ**

agent သည် တိကျသည့် လေယာဉ်ခရီးတစ်ခုကို ဘွတ်ကင်ရန် ကြိုးပမ်းခဲ့သော်လည်း ကန့်သတ်ချက်များကြောင့် မအောင်မြင်ခဲ့ပါက episodic မှတ်ဉာဏ်သည် ဤမအောင်မြင်မှုကို မှတ်တမ်းတင်ကာ နောက်တစ်ကြိမ် စမ်းသပ်ရာတွင် အခြားလေယာဉ်များကို စမ်းကြည့်ရန် သို့မဟုတ် အသုံးပြုသူအား ပို၍ အသိပညာရှိစွာ ပြောကြားရန် အကြောင်းပြချက်ပေးပို့နိုင်သည်။

#### Entity မှတ်ဉာဏ်

ဤသည်မှာ စကားဝိုင်းများမှ လူများ၊ နေရာများ သို့မဟုတ် အရာများကဲ့သို့ အထူးသော entity များနှင့် ဖြစ်ရပ်များကို ရှာထုတ်၍ မှတ်ထားသည့် အချက်အလက်များကို ပါဝင်သည်။ ၎င်းက agent ကို ဆွေးနွေးခဲ့သည့် အဓိက အစိတ်အပိုင်းများကို ဖွဲ့စည်းထားသည့် နားလည်မှုတစ်ခု ဖန်တီးရန် အထောက်အကူပြုသည်။

**Entity မှတ်ဉာဏ် ဥပမာ**

အကောင်၏ စကားဝိုင်းတစ်ခုမှ "Paris", "Eiffel Tower", "Le Chat Noir restaurant တွင် ညစာစားခြင်း" ကဲ့သို့သော entity များကို ရောင်းထုတ်နိုင်သည်။ အနာဂတ် ဆက်ဆံမှုတစ်ခုတွင် agent သည် "Le Chat Noir" ကို ပြန်လည် သတိရကာ ထိုနေရာတွင် နောက်တစ်ဆက် réservation ပြုလုပ်ပေးမည်ဟု အကြံပြုနိုင်သည်။

#### စနစ်တကျ ဖွဲ့စည်းထားသော RAG (Structured RAG)

RAG သည် ကျယ်ပြန့်သည့် နည်းလမ်းတစ်ခုဖြစ်သော်လည်း "Structured RAG" ကို အင်အားကောင်းသော မှတ်ဉာဏ်နည်းပညာအဖြစ် အထူးဖော်ပြထားသည်။ ၎င်းသည် စကားဝိုင်းများ၊ အီးမေးလ်များ၊ ပုံများကဲ့သို့ မတူညီသည့် အရင်းအမြစ်များမှ ထူထပ်ပြီး ဖွဲ့စည်းထားသည့် သတင်းအချက်အလက်များကို ချဲ့ထည်၍ တိကျမှု၊ ပြန်လည်ရရှိနိုင်မှုနှင့် အမြန်နှုန်းကို တိုးမြှင့်ပေးသည်။ စာသားအပိုင်းများ၏ semantic ဆင်တူမှု အပေါ်သာ မူတည်သော ရိုးရာ RAG နည်းလမ်းမှ ကွဲ၍၊ Structured RAG သည် သတင်းအချက်အလက်၏ တည်ရှိမှုဖွဲ့စည်းမှုနှင့် အလုပ်လုပ်သည်။

**Structured RAG ဥပမာ**

စကားလုံးများကိုသာ ကိုက်ညီခြင်းမပြုဘဲ Structured RAG သည် အီးမေးလ်မှ လေယာဉ်အသေးစိတ်များ (destination, date, time, airline) ကို ဖော်ထုတ်၍ ဖွဲ့စည်းထားသော ပုံစံဖြင့် သိမ်းဆည်းနိုင်သည်။ ဤကဲ့သို့ သိမ်းဆည်းမှုဖြင့် "ကျွန်ုပ်က မနေ့အင်္ဂါနေ့တွင် Paris သို့ ဘယ်လေယာဉ်ကို ဘွတ်ကင်ခဲ့ပါသလဲ?" ကဲ့သို့ တိကျစွာ မေးမြန်းနိုင်သည်။

## မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်ခြင်းနှင့် သိမ်းဆည်းခြင်း

AI အေဂျင့်များအတွက် မှတ်ဉာဏ်ကို အကောင်အထည်ဖော်ခြင်းဆိုသည်မှာ memory management ဆိုသော စနစ်တကျဖြစ်စဉ်တစ်ခုကို လိုအပ်သည် — ၎င်းတွင် အချက်အလက်များ ဖန်တီးခြင်း၊ သိမ်းဆည်းခြင်း၊ ရှာဖွေရေး၊ ပေါင်းစပ်ခြင်း၊ အပ်ဒိတ်လုပ်ခြင်း နှင့် "မေ့သည်" (သို့) ဖျက်ပစ်ခြင်းများ ပါဝင်သည်။ ရှာဖွေရေး (retrieval) သည် အထူးသဖြင့် အရေးပါသည်။

### အထူးပြုမှတ်ဉာဏ် ကိရိယာများ

#### Mem0

agent မှတ်ဉာဏ်ကို သိမ်းဆည်းစီမံရန် အသုံးပြုနိုင်သော ပထမတန်းကိရိယာများထဲမှ တစ်ခုမှာ Mem0 ဖြစ်သည်။ Mem0 သည် အမြဲတမ်း နေရာယူသော မှတ်ဉာဏ်အလွှာအဖြစ် လည်ပတ်ပြီး agent များအား သက်ဆိုင်ရာ ဆက်ဆံမှုများကို ပြန်လည် မှတ်ယူပေးခြင်း၊ အသုံးပြုသူနှစ်သက်ချက်များနှင့် အချက်အလက်ဘက်ဒေတာများကို သိမ်းဆည်းပေးခြင်းနှင့် အချိန်အတိုင်းအတာကြာလာသည်နှင့် အမျှ အောင်မြင်မှုများနှင့် မအောင်မြင်မှုများမှ သင်ယူစေသည်။ အတွေးမှာ stateless agents များကို stateful agents များဖြစ်လာစေခြင်းဖြစ်သည်။

၎င်းသည် **နှစ်ချက်ဖြင့် လုပ်ဆောင်သည့် မှတ်ဉာဏ် လိုင်းဧကလမ်းဖြစ်စဉ် — ခွဲထုတ်ခြင်းနှင့် အပ်ဒိတ်ခြင်း** ဖြင့် လည်ပတ်သည်။ ပထမဦးဆုံး agent ၏ thread ထဲသို့ ထည့်သွင်းသော မက်ဆေ့ချ်များကို Mem0 ဝန်ဆောင်မှုပို့ပေးပြီး ၎င်းမှာ LLM ကို အသုံးပြုကာ စကားဝိုင်းသမိုင်းကို အကျဉ်းချုပ်ပေးပြီး မှတ်ဉာဏ်အသစ်များကို ခွဲထုတ်သည်။ ထို့နောက် LLM ဖြင့် အပ်ဒိတ်လုပ်ငန်းစဉ်သည် ဤမှတ်ဉာဏ်များကို ထည့်ရန်၊ ပြင်ဆင်ရန် သို့မဟုတ် ဖျက်ရန် ဆုံးဖြတ်ပေးပြီး ဗက်တာ၊ ဂရပ်နှင့် key-value ဒေတာဘေ့စ်များ ပါဝင်နိုင်သည့် hybrid ဒေတာစတိုးထဲသို့ သိမ်းဆည်းပေးသည်။ ဤစနစ်သည် မျိုးစုံသော မှတ်ဉာဏ်အမျိုးအစားများကို ထောက်ပံ့ပြီး entities များအကြားဆက်နွယ်မှုများကို စီမံရန် graph memory ကိုလည်း ထည့်သွင်းနိုင်သည်။

#### Cognee

အားကောင်းသည့် နည်းလမ်းတစ်ခုမှာ **Cognee** ဖြစ်ပြီး ၎င်းသည် AI အေဂျင့်များအတွက် open-source semantic memory ဖြစ်သည်။ Cognee သည် ဖွဲ့စည်းထားသောနှင့် မဖွဲ့စည်းထားသော ဒေတာများကို embeddings ဖြင့် ထောက်ပံ့ထားသည့် query ပြုလို့ရသော knowledge graphs များသို့ ပြောင်းလဲပေးသည်။ Cognee သည် vector similarity search နှင့် graph relationship များကို ပေါင်းစပ်သည့် **dual-store architecture** ကို ပေးဆောင်၍ agent များအား သတင်းအချက်အလက်များသည် မည်သည့်အရာနှင့် ဆင်တူကြောင်းသာမက အယူအဆများသည် အချင်းချင်း မည်သို့ ဆက်စပ်ကြသည်ကို သိရှိစေရန် ခွင့်ပြုသည်။

၎င်းသည် vector similarity, graph structure, နှင့် LLM မှ ချဉ်းကပ်ချက်များကို ပေါင်းစည်းသည့် **hybrid retrieval** တွင် ထူးချွန်ပြီး - မူရင်း chunk lookup မှ graph-aware question answering အထိ အမျိုးမျိုးကို ထောက်ပံ့သည်။ စနစ်သည် တိုးတက်နေသည့် **living memory** ကို ထိန်းသိမ်းပြီး တစ်ခုချင်းဆက်စပ် ဂရပ်အဖြစ် ရှိနေဆဲဖြစ်စေသည်၊ session အတိုချုံး context နှင့် ရေရှည်တည်ငြိမ်သော မှတ်ဉာဏ်တို့ကို ကူညီပေးသည်။

Cognee notebook သင်ခန်းစာ ([13-agent-memory-cognee.ipynb](./13-agent-memory-cognee.ipynb)) တွင် ဤသဘောထားတစ်ခုကို ဖန်တီးခြင်းကို ပြသပြီး မတူညီသော ဒေတာအရင်းအမြစ်များကို ingest ပြုလုပ်ခြင်း၊ knowledge graph ကို မြင်ကွင်းဖော်ပြခြင်းနှင့် agent တစ်ခုချင်းစီ၏ လိုအပ်ချက်အလိုက် ရှာဖွေရေးနည်းလမ်းပေါင်းများကို အသုံးပြု၍ query ချခြင်းတို့၏ လက်တွေ့ ဥပမာများကို ဖော်ပြထားသည်။

### RAG ဖြင့် မှတ်ဉာဏ် သိမ်းဆည်းခြင်း

mem0 ကဲ့သို့ အထူးပြု မှတ်ဉာဏ် ကိရိယာများအပြင်၊ သင်သည် သိမ်းဆည်းမှုများကို သိမ်းထားရန်နှင့် ပြန်လည်ရှာဖွေရန် အတွက် **Azure AI Search** ကဲ့သို့သော စွမ်းအားရှိသော search ဝန်ဆောင်မှုများကို backend အဖြစ် အသုံးချနိုင်သည်၊ အထူးသဖြင့် structured RAG အတွက်။

၎င်းက သင့် agent ၏ ဖြေရှင်းချက်များကို သင့်ရဲ့ဒေတာနှင့် ချိတ်ဆက်ထားစေပြီး ပိုမို သက်ဆိုင်ရာနှင့် မှန်ကန်သော ဖြေရှင်းချက်များ ပေးနိုင်စေသည်။ Azure AI Search ကို အသုံးပြု၍ အသုံးပြုသူအထူး ခရီးစဉ်မှတ်ဉာဏ်များ၊ ထုတ်ကုန်စာရင်းများ သို့မဟုတ် မည်သည့် domain အထူး ဗဟုတဏ်လည်း ဖြစ်နိုင်သည့် သုတေသနများကို သိမ်းဆည်းနိုင်သည်။

Azure AI Search သည် **Structured RAG** ကဲ့သို့သော လက္ခဏာများကို ထောက်ပံ့ပြီး စကားဝိုင်းသမိုင်းများ၊ အီးမေးလ်များ သို့မဟုတ် ပုံများကဲ့သို့ ကြီးမားသော ဒေတာစုစည်းမှုများမှ ထူထပ်၊ ဖွဲ့စည်းထားသည့် သတင်းအချက်အလက်များကို ချဲ့ထည့်ပြီး ရှာဖွေရေးနှင့် တုံ့ပြန်ခြင်းတို့တွင် "လူ့စွမ်းရည်ထက် ထိရောက်မှုနှင့် ပြန်လည်ရရှိနိုင်မှု" ကို ပေးနိုင်သည်။

## AI အေဂျင့်များကို ကိုယ်တိုင်တိုးတက်စေခြင်း

ကိုယ်တိုင်တိုးတက်လာနိုင်သည့် agent များအတွက် အထူး pattern တစ်ခုမှာ **"knowledge agent"** ကို မိတ်ဆက်သည့်အခြေအနေဖြစ်သည်။ ဤ သက်ဆိုင်ရာ agent သည် အသုံးပြုသူနှင့် အဓိက agent အကြား ဖြစ်ပွားသည့် ဆက်ဆံမှုကို ကြည့်ရှုသည့် သီးခြား agent တစ်ယောက် ဖြစ်သည်။ ၎င်း၏တာဝန်များမှာ -

1. **တန်ဖိုးရှိသော သတင်းအချက်အလက်များကို ဖော်ထုတ်ခြင်း**: စကားဝိုင်းရှိ အပိုင်းတစ်ပိုင်းက ယခုနှင့်သက်ဆိုင်၍ ယာယီ သိမ်းရန် သို့မဟုတ် အထွေထွေ သဘောတရားအဖြစ် သိမ်းဆည်းရန် တန်ဖိုးရှိမရှိ ကောက်ချက်ချသည်။

2. **ဖော်ထုတ်၍ အကျဉ်းချုပ်ခြင်း**: ဆက်ဆံမှုမှ အဓိကသင်ယူချက် သို့မဟုတ် နှစ်သက်ချက်ကို ချုပ်ချယ်သည်။

3. **knowledge base တွင် သိမ်းဆည်းခြင်း**: ပုံမှန်အားဖြင့် ဗက်တာဒေတာဘေ့စ်တစ်ခုအဖြစ် ၎င်းဖော်ထုတ်ထားသည့် အချက်အလက်ကို သိမ်းဆည်းသည်၊ ထိုအချက်များကို နောက်ပိုင်း ရှာဖွေရန် အသုံးပြုနိုင်သည်။

4. **အနာဂတ်မေးခွန်းများကို တိုးချဲ့ပေးခြင်း**: အသုံးပြုသူက မေးခွန်းအသစ် တင်ပြသောအခါ knowledge agent သည် သက်ဆိုင်ရာ သိမ်းဆည်းထားသော အချက်များကို ခေါ်ယူ၍ အသုံးပြုသူ၏ prompt သို့ ပူးပေါင်းထည့်ပေးကာ အဓိက agent ကို အရေးကြီးသော context ပေးစေသည် (RAG ဆန်ဆန်)။

### မှတ်ဉာဏ်အတွက် အရှုပ်တော်ပုံချိန်ညှိချက်များ

• **လက်မလှည့်ချိန်စီမံခြင်း (Latency Management)**: အသုံးပြုသူ ဆက်ဆံမှုများကို မယ်လော့ခဏသည် အလိုမဲ့အားဖြင့် တိုးချဲ့မဖြစ်စေရန် အရင်ဆင်သည့် အချောပိုမို မြန်၍ ကျပြီးစျေးနူန်းချိုမြင့်သော မော်ဒယ်တစ်ခုကို စတင်အသုံးပြု၍ အချက်အလက်ကို သိမ်းဆည်းရန် သို့မဟုတ် ပြန်လည်ခေါ်ယူရန် တန်ဖိုးရှိမရှိ ကို မြန်မြန်စစ်ဆေးနိုင်ပြီး လိုအပ်လျှင်သာ ပို၍ ကြီးကျယ်သော ခွဲထုတ်/ထုတ်ယူလုပ်ငန်းစဉ်များကို ခေါ်ယူနိုင်သည်။

• **Knowledge Base ထိန်းသိမ်းမှု**: knowledge base တိုးတက်လာသည်နှင့်အမျှ နည်းမကြာခဏ အသုံးပြုသော အချက်အလက်များကို ပူဖောင်းတွင် ထားပြီး အသုံးမများသော အချက်အလက်များကို "cold storage" သို့ ရွှေ့ပေးကာ ကုန်ကျစရိတ်ကို ထိန်းချုပ်နိုင်သည်။

## Agent မှတ်ဉာဏ်အကြောင်း မေးစရာများ ပိုမိုရှိပါသလား?
