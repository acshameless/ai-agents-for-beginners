# Quality Checklist

Before you deliver notebook:

- Run am from top to bottom at least one time (or as environment fit allow).
- Make sure say early cells set all the needed state; no carry hidden state from before runs.
- Keep outputs clean. No make output big if short summary fit work.
- Make small tables, key metrics, or short printouts dey.
- Make the story easy to read. Use headings and short bullets, no long paragraphs.
- Only leave helpful TODOs if e necessary, and mark dem well.
- If e no fit execute, talk the risk and how to check am for your own machine.
