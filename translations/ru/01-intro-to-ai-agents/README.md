## Определение AI-агентов и типы AI-агентов

### Что такое AI-агенты?

AI-агенты — это **системы**, которые позволяют **моделям большого языка (LLM)** **выполнять действия**, расширяя их возможности, предоставляя LLM **доступ к инструментам** и **знаниям**.

Давайте разберём это определение на более мелкие части:

- **Система** — важно рассматривать агентов не как один компонент, а как систему из многих компонентов. На базовом уровне, компоненты AI-агента:
  - **Окружение** — определённое пространство, в котором работает агент. Например, если у нас есть AI-агент по бронированию путешествий, окружением может быть система бронирования, которой агент пользуется для выполнения задач.
  - **Датчики** — окружение содержит информацию и предоставляет обратную связь. AI-агенты используют датчики для сбора и интерпретации информации о текущем состоянии окружения. В примере агента по бронированию путешествий, система может предоставлять информацию о доступности отелей или ценах на авиабилеты.
  - **Актуаторы** — после того, как агент получил текущее состояние окружения, для текущей задачи он определяет, какое действие выполнить, чтобы изменить окружение. В случае агента по бронированию это может быть бронирование доступного номера для пользователя.

![Что такое AI-агенты?](../../../translated_images/ru/what-are-ai-agents.1ec8c4d548af601a.webp)

**Модели большого языка** — концепция агентов существовала ещё до появления LLM. Преимущество создания AI-агентов на базе LLM заключается в их способности интерпретировать человеческий язык и данные. Эта способность позволяет LLM интерпретировать информацию окружения и определять план по изменению окружения.

**Выполнение действий** — вне системы AI-агентов LLM ограничены задачами генерации контента или информации на запрос пользователя. В системах AI-агентов LLM могут выполнять задачи, интерпретируя запрос пользователя и используя доступные в окружении инструменты.

**Доступ к инструментам** — доступные LLM инструменты определяются 1) окружением, в котором он работает, и 2) разработчиком AI-агента. Для нашего примера с агентом по бронированию, инструменты агента ограничены операциями в системе бронирования, и/или разработчик может ограничить доступ агента к бронированию рейсов.

**Память и знания** — память может быть кратковременной в контексте разговора между пользователем и агентом. В долгосрочной перспективе, помимо информации, предоставляемой окружением, AI-агенты могут извлекать знания из других систем, сервисов, инструментов и даже других агентов. В примере с агентом по бронированию знания могут содержать информацию о предпочтениях пользователя, хранящуюся в базе данных клиентов.

### Различные типы агентов

Теперь, когда у нас есть общее определение AI-агентов, рассмотрим некоторые конкретные типы агентов и как они применимы к агенту по бронированию путешествий.

| **Тип агента**                | **Описание**                                                                                                                       | **Пример**                                                                                                                                                                                                                   |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Простые рефлексные агенты** | Выполняют немедленные действия на основе заранее заданных правил.                                                                 | Агент по путешествиям интерпретирует контекст письма и перенаправляет жалобы на путешествия в службу поддержки.                                                                                                               |
| **Модели-рефлексные агенты**  | Выполняют действия на основе модели мира и изменений в этой модели.                                                                | Агент по путешествиям приоритезирует маршруты с существенными изменениями цен, используя доступ к историческим данным по ценам.                                                                                               |
| **Агенты, ориентированные на цели** | Создают планы для достижения конкретных целей, интерпретируя цель и определяя действия для её выполнения.                        | Агент бронирует путешествие, определяя необходимые транспортные средства (автомобиль, общественный транспорт, авиарейсы) от текущего местоположения до пункта назначения.                                                     |
| **Агенты, ориентированные на полезность** | Учитывают предпочтения и численно оценивают компромиссы для достижения целей.                                                   | Агент по путешествиям максимизирует полезность, взвешивая удобство против стоимости при бронировании поездки.                                                                                                                |
| **Обучающиеся агенты**         | Улучшаются со временем, реагируя на обратную связь и соответствующим образом корректируя действия.                               | Агент по путешествиям совершенствуется, используя отзывы клиентов из опросов после поездок для корректировки будущих бронирований.                                                                                           |
| **Иерархические агенты**       | Содержат несколько агентов в многоуровневой системе, где агенты верхнего уровня делят задачи на подзадачи для агентов нижнего уровня. | Агент по путешествиям отменяет поездку, разделяя задачу на подзадачи (например, отмена конкретных бронирований), которые выполняют нижестоящие агенты, докладывая верхнеуровневому агенту.                                    |
| **Мультиагентные системы (MAS)** | Агенты выполняют задачи автономно, сотрудничая или конкурируя.                                                                    | Сотрудничество: несколько агентов бронируют отдельные услуги (отели, рейсы, развлечения). Конкуренция: несколько агентов управляют и конкурируют за общий календарь бронирования отеля для размещения клиентов.               |

## Когда использовать AI-агентов

В предыдущем разделе мы использовали пример агента по бронированию путешествий, чтобы объяснить, как разные типы агентов применяются в разных сценариях бронирования. Мы продолжим использовать это приложение в течение всего курса.

Давайте рассмотрим задачи, для которых AI-агенты подходят лучше всего:

![Когда использовать AI-агентов?](../../../translated_images/ru/when-to-use-ai-agents.54becb3bed74a479.webp)

- **Задачи с открытым концом** — дают LLM возможность самостоятельно определить необходимые шаги для выполнения задачи, так как не всегда можно жёстко запрограммировать их в рабочий процесс.
- **Многошаговые процессы** — задачи сложной природы, требующие от AI-агента использовать инструменты или информацию в нескольких циклах, а не одномоментном извлечении.
- **Улучшение со временем** — задачи, в которых агент может со временем улучшать работу, получая обратную связь от окружения или пользователей, чтобы предоставлять более полезные результаты.

Мы рассмотрим дополнительные моменты использования AI-агентов в уроке по созданию надёжных AI-агентов.

## Основы агентных решений

### Разработка агента

Первый шаг в проектировании системы AI-агента — определить инструменты, действия и поведение. В этом курсе мы сосредотачиваемся на использовании **Azure AI Agent Service** для определения наших агентов. Он предлагает функции, такие как:

- Выбор открытых моделей, таких как OpenAI, Mistral и Llama
- Использование лицензированных данных через поставщиков, например, Tripadvisor
- Использование стандартизированных инструментов OpenAPI 3.0

### Агентные шаблоны

Взаимодействие с LLM происходит через подсказки (промты). Учитывая полуавтономный характер AI-агентов, не всегда возможно или необходимо вручную повторно вводить подсказку после изменения окружения. Мы используем **агентные шаблоны**, которые позволяют задавать LLM подсказки на нескольких шагах более масштабируемым образом.

Этот курс разделён на некоторые из популярных в настоящее время агентных шаблонов.

### Агентные фреймворки

Агентные фреймворки позволяют разработчикам реализовывать агентные шаблоны через код. Эти фреймворки предоставляют шаблоны, плагины и инструменты для улучшения взаимодействия AI-агентов. Эти возможности обеспечивают лучшую наблюдаемость и отладку систем AI-агентов.

В этом курсе мы рассмотрим фреймворк AutoGen, основанный на исследованиях, и готовый к производству Agent фреймворк от Semantic Kernel.
