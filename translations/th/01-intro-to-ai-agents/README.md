## การกำหนด AI Agents และประเภทของ AI Agents

### AI Agents คืออะไร?

AI Agents คือ **ระบบ** ที่ช่วยให้ **Large Language Models(LLMs)** สามารถ **ดำเนินการต่างๆ** โดยการขยายความสามารถของพวกมันด้วยการให้ LLMs **เข้าถึงเครื่องมือ** และ **ความรู้**

มาแยกคำจำกัดความนี้เป็นส่วนย่อยๆ:

- **ระบบ** - สำคัญที่จะคิดว่าเอเจนต์ไม่ได้เป็นเพียงส่วนประกอบเดียว แต่เป็นระบบที่ประกอบด้วยหลายส่วน ในระดับพื้นฐาน ส่วนประกอบของ AI Agent คือ:
  - **สภาพแวดล้อม** - พื้นที่ที่กำหนดซึ่ง AI Agent กำลังทำงานอยู่ เช่น ถ้าเรามี AI Agent สำหรับจองการเดินทาง สภาพแวดล้อมอาจเป็นระบบจองการเดินทางที่ AI Agent ใช้เพื่อทำงานให้เสร็จ
  - **เซ็นเซอร์** - สภาพแวดล้อมมีข้อมูลและให้ข้อเสนอแนะ AI Agents ใช้เซ็นเซอร์เพื่อรวบรวมและแปลข้อมูลเกี่ยวกับสถานะปัจจุบันของสภาพแวดล้อม ในตัวอย่าง Travel Booking Agent ระบบจองการเดินทางสามารถให้ข้อมูลเช่น ห้องพักที่ว่างหรือราคาตั๋วเครื่องบิน
  - **อุปกรณ์ปฏิบัติการ** - เมื่อ AI Agent ได้รับสถานะปัจจุบันของสภาพแวดล้อม สำหรับงานปัจจุบัน เอเจนต์จะกำหนดว่าควรดำเนินการอะไรเพื่อเปลี่ยนแปลงสภาพแวดล้อม สำหรับเอเจนต์จองการเดินทาง มันอาจจะเป็นการจองห้องพักที่ว่างสำหรับผู้ใช้

![What Are AI Agents?](../../../translated_images/th/what-are-ai-agents.1ec8c4d548af601a.webp)

**Large Language Models** - แนวคิดของเอเจนต์มีอยู่ก่อนการสร้าง LLMs ข้อได้เปรียบของการสร้าง AI Agents ด้วย LLMs คือความสามารถในการตีความภาษามนุษย์และข้อมูล ความสามารถนี้ช่วยให้ LLMs แปลข้อมูลสภาพแวดล้อมและกำหนดแผนเพื่อเปลี่ยนแปลงสภาพแวดล้อม

**ดำเนินการต่างๆ** - นอกเหนือจากระบบ AI Agent แล้ว LLMs จะถูกจำกัดอยู่ในสถานการณ์ที่การดำเนินการคือการสร้างเนื้อหาหรือข้อมูลตามคำสั่งของผู้ใช้ ในระบบ AI Agent LLMs สามารถทำงานได้โดยการตีความคำขอของผู้ใช้และใช้เครื่องมือที่มีในสภาพแวดล้อมของพวกมัน

**การเข้าถึงเครื่องมือ** - เครื่องมือที่ LLM เข้าถึงได้ถูกกำหนดโดย 1) สภาพแวดล้อมที่มันทำงานอยู่ และ 2) ผู้พัฒนาของ AI Agent สำหรับตัวอย่างเอเจนต์การเดินทาง เครื่องมือของเอเจนต์ถูกจำกัดโดยการดำเนินการที่มีอยู่ในระบบจอง และ/หรือผู้พัฒนาสามารถจำกัดการเข้าถึงเครื่องมือเอเจนต์เกี่ยวกับเที่ยวบินได้

**ความจำ+ความรู้** - ความจำอาจเป็นแบบระยะสั้นในบริบทของการสนทนาระหว่างผู้ใช้กับเอเจนต์ ในระยะยาว นอกเหนือจากข้อมูลที่จัดเตรียมโดยสภาพแวดล้อม AI Agents ยังสามารถดึงความรู้จากระบบอื่น บริการ เครื่องมือ และแม้แต่เอเจนต์อื่นๆ ได้ ด้วยตัวอย่างเอเจนต์การเดินทาง ความรู้นี้อาจเป็นข้อมูลเกี่ยวกับความชอบการเดินทางของผู้ใช้ที่อยู่ในฐานข้อมูลลูกค้า

### ประเภทต่างๆ ของเอเจนต์

เมื่อเรามีคำจำกัดความทั่วไปของ AI Agents แล้ว ให้เรามาดูเอเจนต์ประเภทเฉพาะและวิธีการใช้พวกมันในเอเจนต์การจองการเดินทาง

| **ประเภทเอเจนต์**            | **คำอธิบาย**                                                                                                                        | **ตัวอย่าง**                                                                                                                                                                                                                  |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Simple Reflex Agents**      | ดำเนินการทันทีตามกฎที่กำหนดไว้ล่วงหน้า                                                                                       | เอเจนต์การเดินทางตีความบริบทของอีเมลและส่งต่อคำร้องเรียนเรื่องการเดินทางไปยังฝ่ายบริการลูกค้า                                                                                                                              |
| **Model-Based Reflex Agents** | ดำเนินการตามแบบจำลองของโลกและการเปลี่ยนแปลงในแบบจำลองนั้น                                                                        | เอเจนต์การเดินทางให้ความสำคัญกับเส้นทางที่มีการเปลี่ยนแปลงราคาสำคัญ โดยอิงจากการเข้าถึงข้อมูลราคาประวัติ                                                                                                                  |
| **Goal-Based Agents**         | สร้างแผนเพื่อบรรลุเป้าหมายเฉพาะโดยตีความเป้าหมายและกำหนดการดำเนินการเพื่อไปถึงเป้าหมาย                                        | เอเจนต์การเดินทางจองการเดินทางโดยกำหนดการจัดเตรียมการเดินทางที่จำเป็น (รถยนต์ ขนส่งสาธารณะ เที่ยวบิน) จากตำแหน่งปัจจุบันถึงจุดหมายปลายทาง                                                                              |
| **Utility-Based Agents**      | พิจารณาความชอบและชั่งน้ำหนักผลประโยชน์ทางตัวเลขเพื่อกำหนดวิธีบรรลุเป้าหมาย                                                     | เอเจนต์การเดินทางเพิ่มประโยชน์สูงสุดโดยเปรียบเทียบความสะดวกสบายกับค่าใช้จ่ายเมื่อจองการเดินทาง                                                                                                                            |
| **Learning Agents**           | พัฒนาไปตามกาลเวลาโดยตอบสนองต่อข้อเสนอแนะและปรับการดำเนินการตามนั้น                                                            | เอเจนต์การเดินทางพัฒนาด้วยการใช้ข้อเสนอแนะจากลูกค้าผ่านแบบสำรวจหลังการเดินทางเพื่อตั้งค่าการจองในอนาคต                                                                                                                     |
| **Hierarchical Agents**       | มีเอเจนต์หลายตัวในระบบแบบลำดับชั้น โดยเอเจนต์ระดับสูงจะแบ่งงานเป็นงานย่อยให้เอเจนต์ระดับล่างทำให้แล้วเสร็จ                  | เอเจนต์การเดินทางยกเลิกทริปโดยแบ่งงานเป็นงานย่อย (เช่น ยกเลิกการจองเฉพาะเจาะจง) และให้เอเจนต์ระดับล่างทำงานเหล่านั้น พร้อมรายงานกลับไปยังเอเจนต์ระดับสูง                                                                            |
| **Multi-Agent Systems (MAS)** | เอเจนต์ทำงานให้เสร็จอย่างอิสระ ทั้งแบบร่วมมือหรือแข่งขัน                                                                          | ร่วมมือ: เอเจนต์หลายตัวจองบริการเดินทางเฉพาะ เช่น โรงแรม เที่ยวบิน และความบันเทิง แข่งขัน: เอเจนต์หลายตัวจัดการและแข่งขันกันบนปฏิทินจองโรงแรมที่ใช้ร่วมกันเพื่อจองลูกค้าเข้าพักโรงแรม                                      |

## เมื่อไรควรใช้ AI Agents

ในส่วนก่อนหน้า เราใช้กรณีใช้งาน Travel Agent เพื่ออธิบายว่าเอเจนต์ประเภทต่างๆ สามารถใช้ในสถานการณ์การจองเดินทางที่แตกต่างกันอย่างไร เราจะใช้แอปพลิเคชันนี้ตลอดหลักสูตร

มาดูประเภทของกรณีใช้งานที่เหมาะสมที่สุดสำหรับ AI Agents:

![When to use AI Agents?](../../../translated_images/th/when-to-use-ai-agents.54becb3bed74a479.webp)


- **ปัญหาที่เปิดกว้าง** - อนุญาตให้ LLM กำหนดขั้นตอนที่จำเป็นในการทำงานให้เสร็จ เพราะไม่สามารถกำหนดโค้ดล่วงหน้าได้ตลอดเวลาในเวิร์กโฟลว์
- **กระบวนการหลายขั้นตอน** - งานที่ต้องมีความซับซ้อนในระดับที่ AI Agent ต้องใช้เครื่องมือหรือข้อมูลหลายรอบแทนที่จะเป็นการดึงข้อมูลครั้งเดียว  
- **ปรับปรุงตามกาลเวลา** - งานที่เอเจนต์สามารถพัฒนาด้วยการรับข้อเสนอแนะจากสภาพแวดล้อมหรือผู้ใช้ เพื่อให้บริการได้ดีขึ้น

เราจะครอบคลุมการพิจารณาเพิ่มเติมเกี่ยวกับการใช้ AI Agents ในบทเรียนการสร้าง AI Agents ที่น่าเชื่อถือ

## พื้นฐานของโซลูชันแบบ Agentic

### การพัฒนาเอเจนต์

ขั้นตอนแรกในการออกแบบระบบ AI Agent คือการกำหนดเครื่องมือ การดำเนินการ และพฤติกรรม ในหลักสูตรนี้ เราเน้นการใช้ **Azure AI Agent Service** เพื่อกำหนดเอเจนต์ของเรา ซึ่งมีฟีเจอร์เช่น:

- การเลือกใช้โมเดลเปิด เช่น OpenAI, Mistral และ Llama
- การใช้ข้อมูลที่ได้รับอนุญาตจากผู้ให้บริการ เช่น Tripadvisor
- การใช้เครื่องมือ OpenAPI 3.0 ที่เป็นมาตรฐาน

### รูปแบบแบบ Agentic

การสื่อสารกับ LLM เป็นผ่านพรอมต์ เนื่องจากลักษณะกึ่งอัตโนมัติของ AI Agents จึงไม่สามารถหรือไม่จำเป็นต้องให้พรอมต์กับ LLM ใหม่ด้วยตนเองทุกครั้งเมื่อมีการเปลี่ยนแปลงสภาพแวดล้อม เราใช้ **รูปแบบแบบ Agentic** ซึ่งอนุญาตให้เราพรอมต์ LLM หลายขั้นตอนในรูปแบบที่ขยายได้ง่ายขึ้น

หลักสูตรนี้แบ่งออกเป็นรูปแบบแบบ Agentic ยอดนิยมบางส่วนในปัจจุบัน

### กรอบงานแบบ Agentic

กรอบงานแบบ Agentic ช่วยให้นักพัฒนาสามารถใช้รูปแบบแบบ Agentic ผ่านโค้ด กรอบงานเหล่านี้มีเทมเพลต ปลั๊กอิน และเครื่องมือเพื่อการทำงานร่วมกันของ AI Agent ได้ดีขึ้น ข้อดีเหล่านี้มอบความสามารถสำหรับการสังเกตการณ์และแก้ไขปัญหาระบบ AI Agent ได้ดียิ่งขึ้น

ในหลักสูตรนี้ เราจะสำรวจกรอบงาน AutoGen ที่ขับเคลื่อนด้วยการวิจัย และกรอบงาน Agent ที่พร้อมสำหรับการผลิตจาก Semantic Kernel
