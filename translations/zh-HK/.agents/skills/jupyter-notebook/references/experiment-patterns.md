# 實驗模式

針對探索性和實驗性工作，使用此結構：

- 標題與目標：說明問題及成功標準。
- 設定與可重現性：只匯入所需的套件，盡早設定隨機種子，並將設定保存在一個簡短的儲存格中。
- 計劃：在執行程式碼前列出假設、搜尋範圍（sweeps）和指標。
- 最小基準：從最小可執行範例開始，並在增加複雜度前確認其能端到端執行。
- 結果與註記：在相關程式碼附近以 Markdown 摘要發現，並將關鍵指標記錄於小型字典或類表格結構中。
- 下一步：決定是繼續、轉向還是停止，並以簡短要點記下後續想法。
