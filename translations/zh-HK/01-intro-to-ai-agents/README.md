## 定義 AI Agents 及 AI Agents 的種類

### 什麼是 AI Agents？

AI Agents 是一種系統，讓大型語言模型（LLMs）能夠執行動作，藉由賦予 LLMs 使用工具與知識的能力，來擴展它們的功能。

我們把這個定義拆解成幾部分：

- **系統** — 很重要的是，不要只把代理視為一個單一元件，而是一個包含多個元件的系統。在基本層面，AI Agent 的元件有：
  - **環境** — AI Agent 運作的定義空間。例如：旅行訂票 AI Agent 的環境可以是用於完成任務的旅行訂票系統。
  - **感測器** — 環境中具有資訊並提供反饋。AI Agent 使用感測器收集並解讀環境目前狀態的資訊。在旅行訂票代理示例中，訂票系統可提供例如飯店房態或航班價格等資訊。
  - **執行器** — 當 AI Agent 收到環境當前狀態後，會判斷目前任務該執行什麼動作以改變環境。對於旅行訂票代理而言，可能是為使用者預訂一間可用房。

![What Are AI Agents?](../../../translated_images/zh-HK/what-are-ai-agents.1ec8c4d548af601a.webp)

**大型語言模型** — 代理的概念早於 LLM 出現。透過 LLM 建立 AI Agents 的優勢，是它們具備理解人類語言和資料的能力。此能力使 LLM 能解讀環境資訊，並制定改變環境的計畫。

**執行動作** — 在 AI Agent 系統外，LLM 的動作通常限於根據用戶提示產生內容或資訊。在 AI Agent 系統中，LLM 能解析用戶的請求，並利用環境中的工具完成任務。

**使用工具的權限** — LLM 可使用的工具取決於 1) 它所在的運行環境，及 2) AI Agent 開發者的設定。以旅行代理為例，代理可使用的工具受限於訂票系統提供的操作，且開發者也可以限制代理只能使用航班工具。

**記憶與知識** — 記憶可以是用戶與代理對話中的短期記憶。長期來說，除環境提供的資訊外，AI Agents 也能從其他系統、服務、工具甚至其他代理取得知識。在旅行代理中，這些知識可能是存於客戶資料庫中使用者的旅遊偏好。

### 不同種類的 AI Agent

現在我們已經有 AI Agents 的一般定義，接著看看一些特定代理種類，以及它們如何應用在旅行訂票 AI Agent 中。

| **代理種類**                  | **說明**                                                                                                                         | **範例**                                                                                                                                                                                                                 |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **簡單反射代理(Simple Reflex Agents)**      | 根據預定義規則執行即時行動。                                                                                                     | 旅行代理解讀電子郵件內容並將旅遊投訴轉交給客服部門。                                                                                                                                                                     |
| **基於模型反射代理(Model-Based Reflex Agents)** | 根據世界模型及其變化執行行動。                                                                                                    | 旅行代理根據歷史價格數據，優先處理價格變動顯著的路線。                                                                                                                                                                   |
| **目標導向代理(Goal-Based Agents)**         | 通過解讀目標並決定達成目標的行動來建立計劃。                                                                                     | 旅行代理制定計劃以安排行程，如從當前地點至目的地所需的交通（汽車、大眾交通、航班）安排。                                                                                                                             |
| **效用導向代理(Utility-Based Agents)**      | 考慮偏好並以數值方式權衡取捨以決定如何達成目標。                                                                                  | 旅行代理考量便利性與價格，權衡以最大化效用來預訂行程。                                                                                                                                                                   |
| **學習型代理(Learning Agents)**           | 透過回饋學習並調整行為持續改進。                                                                                                | 旅行代理利用出行後調查的顧客反饋，改進未來訂票安排。                                                                                                                                                                   |
| **分層代理(Hierarchical Agents)**       | 多層級代理系統，高階代理將任務拆解成子任務交由低階代理執行。                                                                       | 旅行代理透過將取消行程任務拆成多個子任務（例如逐項取消各預訂），並由低階代理完成，最後回報高階代理。                                                                                                                     |
| **多代理系統 (Multi-Agent Systems, MAS)** | 多重代理各自完成任務，可以合作或競爭。                                                                                             | 合作：多個代理分別預訂旅館、航班及娛樂行程。競爭：多個代理管理並爭奪共享的旅館訂房日曆，為客戶爭取訂房。                                                                                                               |

## 何時使用 AI Agents

在前述章節，我們以旅行代理案例說明不同代理類型如何用於旅行訂票的不同場景。在整個課程中，我們會持續使用這個範例。

以下是 AI Agents 最適合的使用情境：

![When to use AI Agents?](../../../translated_images/zh-HK/when-to-use-ai-agents.54becb3bed74a479.webp)


- **開放式問題** — 允許 LLM 自行決定完成任務所需步驟，因為這些步驟無法全數用硬編碼在工作流程中。
- **多步驟流程** — 需要複雜層次的任務，AI Agent 必須在多個階段利用工具或資訊，而非一次檢索就完成。  
- **持續改善** — 任務隨時間改進，代理可藉由接受環境或使用者回饋調整行為以提供更佳效用。

我們會在「建立值得信賴的 AI Agents」課程中探討更多使用 AI Agents 的考量。

## 智慧代理解決方案基礎

### 代理開發

設計 AI Agent 系統的第一步是定義工具、動作和行為。本課程聚焦於使用 **Azure AI Agent Service** 來定義我們的代理。它提供的功能包括：

- 選擇 Open Models，如 OpenAI、Mistral 與 Llama
- 使用授權資料，透過 Tripadvisor 等提供者
- 採用標準化的 OpenAPI 3.0 工具

### 代理模式

與 LLM 的溝通是透過 prompt。鑑於 AI Agents 半自動的特性，環境改變後並非總需或可能手動重新提示 LLM。我們使用 **Agentic Patterns**，讓我們能夠在多個步驟中更具擴展性地對 LLM 發出提示。

本課程覆蓋現今一些熱門的智慧代理模式。

### 代理框架

代理框架讓開發者透過程式碼實作智慧代理模式，這些框架提供範本、外掛以及工具，利於促進 AI Agents 更佳的協作。這些優勢有助於改善 AI Agent 系統的可觀察性及故障排除。

本課程將探索基於研究的 AutoGen 框架及可投入生產的 Semantic Kernel Agent 框架。
