# 品質檢查清單

Before delivering a notebook:

- 至少由頭到尾執行一次（或在環境允許的情況下盡可能執行）。
- 確保前面的儲存格設置所有必要的狀態；避免來自先前執行的隱藏狀態。
- 保持輸出整潔。當簡短摘要足夠時，避免產生龐大的輸出。
- 優先使用小型表格、關鍵指標或簡短的輸出。
- 讓說明易於略讀。使用標題與簡短的要點，避免冗長段落。
- 僅在必要時留下有用的 TODOs，並清楚標示它們。
- 若無法執行，說明風險並說明如何在本地驗證。
